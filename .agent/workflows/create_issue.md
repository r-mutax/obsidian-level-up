---
description: 仕様を固めてGitHub Issueを作成する手順
---

1. **仕様の相談（Discussion）**
   - ユーザーとエージェントで機能を議論し、要件を固める。
   - 決まった仕様は箇条書きなどで明確にする。

2. **タスク分解**
   - 議論した内容を、実装可能なタスクレベルまで分解する。
   - 手順の依存関係や、修正すべきファイルの見当をつける。

3. **Issue案の作成と提示**
   - 以下のフォーマットでIssueのドラフトを作成し、ユーザーに提示する。
   ```
   Title: [機能名] の実装
   Body:
   # 概要
   [機能の目的と概要]
   
   # タスク
   - [ ] タスク1
   - [ ] タスク2
   
   # 完了条件
   - [条件1]
   ```

4. **Issueの作成**
   - ユーザーの承認を得てからコマンドを実行する。
// turbo
   - `gh issue create --title "[Title]" --body "[Body]"`

5. **タスクリストへの反映**
   - 作成されたIssue番号を `task.md` に追記する。
   - 例: `- [ ] [機能名] <!-- id: XX -->`
